# INDRASNET_SOURCE_TIMELINE_V0_1

Status: Public source timeline  
Scope: Collected public development line from early Indra's Net language to DeciRepo genesis formalization

---

## 1. Purpose

This note records the currently assembled public timeline behind the IndrasNet -> DeciRepo -> DLX development line.

It is narrower than a full historical archive.
Its purpose is to capture:

- major dated public signals,
- their role in the conceptual transition,
- and the current archival status of each source layer.

This note distinguishes between:

- directly published public protocol artifacts,
- public notes and references now preserved in this repository,
- and earlier public post material currently recorded from collected descriptions but not yet fully attached as direct URL archive.

---

## 2. Status Labels

Each entry is marked by dominant status:

- `METAPHOR` — conceptual language, trust metaphor, network intuition
- `ARCHITECTURE` — explicit system design direction, nodes / DAO / product language
- `PROTOCOL` — public formalization, governance surface, protocol framing
- `EVIDENCE` — machine-checkable artifacts, recomputation, root-of-trust references

Archival confidence labels:

- `DIRECT` — directly represented by current public files or public URLs
- `ARCHIVED` — preserved in this repository as collected text and image description, even if original platform URL or screenshot is not yet attached
- `RECORDED` — described from collected public material, but direct source URL or screenshot not yet attached in this repository

## 3. 2022-2023 Conceptual Layer

### 2022-2023 — early Indra's Net framing

Status:

- Layer: `METAPHOR`
- Archive confidence: `RECORDED`

Recorded public theme:

- each node reflects the whole,
- rupture or falsehood at one node affects the network,
- and truth across a network cannot be treated as merely local.

Why it matters:

- this is the earliest known public trust intuition behind the later architectural work,
- and it introduces the problem that later becomes protocol language:
  distortion at one node should not silently propagate through the network.

Current archive note:

- exact post URLs / screenshots for this period are not yet attached here,
- but the conceptual line is preserved in later continuity notes.
